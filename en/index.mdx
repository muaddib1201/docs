# Pantum SDK Documentation

Welcome to the Pantum SDK documentation site.

The English section is organized to match the way most teams adopt the SDK in practice:

- Start with the integration guides
- Use the example projects to get a working setup quickly
- Move into the API docs when you need endpoint-level detail
- Use the FAQ and changelog for troubleshooting and release tracking

This keeps the onboarding path short for first-time integrators while still making the API reference easy to use during implementation and debugging.

## Recommended Reading Path

### Start with the Integration Guides

If you are new to the SDK, begin here:

- [Overview](/en/getting-started/overview)
- [Quickstart](/en/getting-started/quickstart)

### Then Open the Example Projects

If you want a platform-specific setup reference, continue with:

- [Example Projects](/en/getting-started/examples)
- [Android Demo](/en/getting-started/examples/android)
- [OpenHarmony Demo](/en/getting-started/examples/openharmony)
- [Domestic OS Demo](/en/getting-started/examples/domestic)

### Then Move into the API Docs

Once the service is running, use the API docs for day-to-day integration work:

- [Usage Overview](/en/api-reference/overview)
- [Core APIs](/en/api-reference/core/service-info)
- [Print APIs](/en/api-reference/print/print-jobs)
- [Scan APIs](/en/api-reference/scan/scan-jobs)
- [PDF APIs](/en/api-reference/pdf/pdf-create)
- [Device Management APIs](/en/api-reference/device/device-status)

## Current Documentation Structure

1. Getting Started: for teams integrating the SDK for the first time
2. Example Projects: for platform-specific setup and validation
3. API Docs: for implementation, debugging, and response lookup
4. Resources: for FAQs, troubleshooting notes, and changelog updates
