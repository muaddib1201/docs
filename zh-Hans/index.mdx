# Pantum SDK 文档

欢迎来到 Pantum SDK 文档站。

Pantum SDK 当前建议先按设备类型进入，再细分到具体操作系统：

- 移动设备：Android、开源鸿蒙
- PC 平台：Windows、国产系统

这样的组织方式更符合 SDK 文档的实际查找路径：开发者通常会先确认项目是运行在移动设备还是 PC 平台，再进一步确认具体操作系统，最后进入安装、初始化、设备发现、打印任务等能力文档。

## 推荐阅读路径

### 先确认平台

如果你还在梳理方案，先看 [平台总览](/platforms/overview)。

如果你已经明确设备类型，建议从一级平台页进入：

- [移动设备 - 概览](/platforms/mobile)
- [PC 平台 - 概览](/platforms/pc)

如果你已经明确具体系统，也可以直接进入对应页面：

- [Android](/platforms/android)
- [开源鸿蒙](/platforms/openharmony)
- [Windows](/platforms/windows)
- [国产系统](/platforms/domestic)

### 再完成基础接入

从安装、初始化、首个打印任务开始，先跑通一条最短路径：

- [安装](/installation/overview)
- [快速开始](/getting-started/quickstart)

### 最后按能力深入

当平台接入跑通后，再进入核心能力与 API 参考：

- [设备发现](/sdk/device-discovery)
- [打印任务](/sdk/print-jobs)
- [状态事件](/sdk/status-events)
- [API 概览](/api-reference/overview)

## 当前文档结构

1. 开始使用：面向第一次接入的开发者
2. 平台接入：面向需要先按设备类型、再按系统定位内容的开发者
3. 核心能力：面向已经完成初始化的开发者
4. API 参考：面向需要查字段、查返回值的开发者
