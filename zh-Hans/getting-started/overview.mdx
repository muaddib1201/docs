## 产品说明

Pantum SDK 以本地 HTTP 服务为统一接入入口，将设备发现、打印、扫描、PDF 预处理与设备管理能力收敛到一套一致的接口体系中，适合在移动端与 PC 端应用中快速完成集成。

<Tip>
  如果你是第一次接入，推荐先阅读 [快速开始](/zh-Hans/getting-started/quickstart)，跑通初始化、服务校验与首次设备发现；如果你需要先判断能力边界，再继续查看下方 API 能力版图。
</Tip>

## 产品优势

<CardGroup cols={2}>
  <Card title="统一接入入口" icon="server">
    所有能力都通过本地 HTTP 服务暴露，启动后统一访问 `http://127.0.0.1:{port}`，便于调试、封装与跨平台复用。
  </Card>
  <Card title="能力链路完整" icon="layer-group">
    从服务信息、支持机型、设备发现，到打印、扫描、PDF 预处理与设备管理，覆盖接入到执行的完整主链路。
  </Card>
  <Card title="面向真实机型能力" icon="sliders">
    多个接口直接围绕 `print_capability`、`scan_capability` 与机型兼容性设计，适合按设备能力动态构造业务参数。
  </Card>
  <Card title="文档结构清晰" icon="compass">
    API 文档已按核心能力域拆分，既适合首次理解体系，也适合在开发过程中按模块快速定位接口。
  </Card>
</CardGroup>

## API 能力版图

当前 API 文档围绕 5 个主能力域展开，外加错误码与 FAQ 等辅助内容，已经形成较完整的接入参考体系。

<CardGroup cols={2}>
  <Card title="核心 API" icon="sparkles" href="/zh-Hans/api-reference/core/service-info">
    7 个接口，负责服务信息读取、支持机型查询、USB 与 NET 设备发现，是所有业务调用的起点。
  </Card>
  <Card title="打印 API" icon="/images/icons/api-print.svg" href="/zh-Hans/api-reference/print/print-jobs">
    7 个接口，覆盖打印作业创建、文件传输、取消、删除、结果查询，以及机型打印能力兼容性确认。
  </Card>
  <Card title="扫描 API" icon="/images/icons/api-scan.svg" href="/zh-Hans/api-reference/scan/scan-jobs">
    5 个接口，提供扫描作业的创建、查询、取消与删除，参数设置依赖机型扫描能力返回值。
  </Card>
  <Card title="PDF API" icon="/images/icons/api-pdf.svg" href="/zh-Hans/api-reference/pdf/pdf-create">
    7 个接口，用于 PDF 创建与预览任务处理，支持页级元数据、BMP 与 RAW 数据获取。
  </Card>
  <Card title="设备管理 API" icon="gear" href="/zh-Hans/api-reference/device/device-status">
    4 个接口，聚焦设备状态、状态列表、SLP 信息与休眠时间配置，面向运行态管理与诊断。
  </Card>
  <Card title="辅助参考" icon="circle-info" href="/zh-Hans/api-reference/error-codes">
    配套提供 [错误码](/zh-Hans/api-reference/error-codes) 与 [常见问题](/zh-Hans/resources/faq)，便于排查初始化、发现设备与任务执行问题。
  </Card>
</CardGroup>

## 推荐阅读路径

<Steps>
  <Step title="确认服务可用">
    从 [查询服务信息](/zh-Hans/api-reference/core/service-info) 或 [快速开始](/zh-Hans/getting-started/quickstart) 开始，先确认 SDK 已启动，本地 HTTP 服务可访问。
  </Step>
  <Step title="读取支持能力">
    阅读 [支持设备列表](/zh-Hans/api-reference/core/supported-devices) 与 [机型接口支持情况（CSV）](/zh-Hans/api-reference/print/print-model-deps-csv)，确认目标机型与功能边界。
  </Step>
  <Step title="发现目标设备">
    根据接入场景进入 [USB设备列表](/zh-Hans/api-reference/core/usb-devices) 或 [开始 NET 设备搜索](/zh-Hans/api-reference/core/net-devices-start)，拿到后续调用所需的设备连接信息。
  </Step>
  <Step title="进入业务能力">
    打印场景继续查看 [创建打印作业](/zh-Hans/api-reference/print/print-job-create)，扫描场景继续查看 [创建扫描作业](/zh-Hans/api-reference/scan/scan-job-create)，PDF 预处理场景进入 [创建预览作业](/zh-Hans/api-reference/pdf/pdf-preview-job-create)。
  </Step>
</Steps>

1. 开始使用：面向第一次接入的开发者，先建立整体认知并跑通最短路径。
